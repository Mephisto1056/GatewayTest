# 数据库表格介绍
1. 数据库分为 question_selfdirected, question_highleveltest, question_mediumleveltest, question_lowleveltest
2. question_selfdirected 表为自主导向
3. question_highleveltest 表为高层专属评分题+开放式问题，有些题可能会有计分规则（反向记分）和相关性（指的是自主导向中的题目编号）
4. question_mediumleveltest 表为中层专属评分题+开放式问题，有些题可能会有计分规则（反向记分）和相关性（指的是自主导向中的题目编号）
5. question_lowleveltest 表为基层专属评分题+开放式问题，有些题可能会有计分规则（反向记分）和相关性（指的是自主导向中的题目编号）

# 不同用户不同问卷
·高层领导者(总经理/总经理-1):自主导向(23道评分题)+高层(40道评分题)+开放式问题(3道)
·中层管理者(总经理-2):自主导向(23道评分题)+中层(36道评分题)+开放式问题(3道)
·基层管理者(一线管理者):自主导向(23道评分题)+基层(33道评分题)+开放式问题(3道)

# 评分规则讲述
评分题，按照1-5分，1=几乎从不，2=很少，3=有时，4=经常，5=几乎总是

反向计分:为避免答题人有被潜意识引导的倾向，某些题目设计为负向行为描述，计分时按照(6-得分)进行计算

相关性:自主导向的测评方向是其他三个面向能力的深层因素，因此自主导向的题目分数会影响其他三个方面的得分。按照(自主导向题目得分)*40% +(目标题目得分)*60% 计算目标题目的得分

360度他评得分计算方式:
高层(总经理-1):上级*50%+平级30%+下级20%
中层(总经理-2):上级*40%+平级40%+下级20%
基层(一线管理):上级*30%+平级30%+下级40%

团队计分规则:以团队每个人的他评得分作为一条数据线，看团队整体能力趋势(开发团队待研究用哪种方式表达更直观。
