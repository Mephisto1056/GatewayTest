# 邮件服务配置指南

您不需要一定要购买额外的邮件服务器。对于开发测试或小规模使用，您有多种免费或低成本的选择。

## 选项一：使用个人邮箱 (最简单，适合测试)

如果您有 Gmail、QQ 邮箱或 163 邮箱，可以直接使用它们作为发送方。

**注意**：为了安全，大多数邮箱不允许直接使用您的登录密码，而是需要生成一个**应用专用密码 (App Password)** 或 **授权码**。

### 1. Gmail 配置示例
*   **HOST**: `smtp.gmail.com`
*   **PORT**: `587`
*   **USER**: 您的 Gmail 地址 (例如 `example@gmail.com`)
*   **PASS**: **应用专用密码** (不是您的 Google 账户登录密码)
    *   *获取方法*: Google 账户设置 -> 安全性 -> 两步验证 -> 应用专用密码 -> 创建一个新的。

### 2. QQ 邮箱配置示例
*   **HOST**: `smtp.qq.com`
*   **PORT**: `465`
*   **USER**: 您的 QQ 邮箱地址 (例如 `123456@qq.com`)
*   **PASS**: **授权码**
    *   *获取方法*: 邮箱设置 -> 账户 -> POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务 -> 开启 POP3/SMTP 服务 -> 生成授权码。

### 3. 163 邮箱配置示例
*   **HOST**: `smtp.163.com`
*   **PORT**: `465`
*   **USER**: 您的 163 邮箱地址
*   **PASS**: **授权码**
    *   *获取方法*: 设置 -> POP3/SMTP/IMAP -> 开启服务 -> 获取授权码。

---

## 选项二：使用专业邮件发送服务 (推荐生产环境)

这些服务专门用于发送系统通知邮件，通常送达率更高，且不易被被当做垃圾邮件。它们通常都有免费额度。

### 1. Brevo (原 Sendinblue)
*   提供每天 300 封免费邮件。
*   注册后获取 SMTP Server, Login, Password。

### 2. SendGrid
*   提供每天 100 封免费邮件。

---

## 如何修改配置

请打开 `backend/.env` 文件，找到以下部分并修改为您的实际信息：

```env
SMTP_HOST=smtp.qq.com      <-- 邮件服务器地址
SMTP_PORT=465              <-- 端口 (通常是 465 或 587)
SMTP_USER=123456@qq.com    <-- 您的邮箱账号
SMTP_PASS=abcdefghijklmn   <-- 您的授权码或应用密码
SMTP_FROM="组织领导力系统" <123456@qq.com>  <-- 发件人显示名称
